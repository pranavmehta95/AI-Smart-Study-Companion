================================================================
        AI SMART STUDY COMPANION - PROJECT DETAILS
================================================================

PROJECT NAME    : AI Smart Study Companion
VERSION         : 1.0.0
DATE            : February 2026
DESCRIPTION     : A full-stack AI-powered web application that helps
                  students study smarter by transforming uploaded notes
                  and PDFs into summaries, quizzes, flashcards, viva
                  practice, and personalized study plans — all powered
                  by Google's Gemini AI.

================================================================
                        TECH STACK
================================================================

FRONTEND:
  - React (Vite) — Component-based UI framework
  - React Router DOM — Client-side routing & navigation
  - Axios — HTTP client for API communication
  - Firebase (Client SDK) — Google OAuth sign-in
  - Recharts — Data visualization / analytics charts
  - CSS (Custom) — Fully custom responsive styling

BACKEND:
  - Node.js + Express.js — REST API server
  - MongoDB + Mongoose — NoSQL database & ODM
  - Firebase Admin SDK — Verifying Google OAuth tokens
  - JSON Web Tokens (JWT) — Authentication & session management
  - bcryptjs — Password hashing & comparison
  - Multer — File upload middleware
  - pdf-parse — PDF text extraction
  - @google/generative-ai — Gemini AI integration
  - dotenv — Environment variable management
  - cors — Cross-Origin Resource Sharing
  - nodemon — Auto-restart during development

================================================================
                    AUTHENTICATION SYSTEM
================================================================

1. EMAIL / PASSWORD REGISTRATION
   - Users register with name, email, and password
   - Passwords are hashed using bcryptjs before storage
   - On success, a JWT token (7-day expiry) is returned

2. EMAIL / PASSWORD LOGIN
   - User submits email and password
   - Password is compared against the hashed DB version
   - JWT token issued on successful login
   - Google-only accounts are blocked from password login
     with a helpful redirect message

3. GOOGLE OAUTH (Firebase)
   - User clicks "Continue with Google" on the frontend
   - Firebase client SDK initiates Google sign-in
   - Firebase ID token is sent to the backend
   - Backend uses Firebase Admin SDK to verify the token
   - If the user exists → linked with Google ID
   - If new user → account created without a password
   - JWT token issued for subsequent API requests

4. JWT MIDDLEWARE
   - All protected routes require a valid JWT in the
     Authorization header: "Bearer <token>"
   - Invalid or missing tokens return 401 Unauthorized

================================================================
                    DOCUMENT MANAGEMENT
================================================================

- Users can upload PDF files or plain text study materials
- Files are stored in the /server/uploads/ directory
- Multer middleware handles the file upload process
- pdf-parse extracts raw text from PDF files automatically
- Extracted text is stored in MongoDB for AI processing
- Supported file types: PDF, TXT
- Max file size: 10 MB
- Users can view their list of uploaded documents
- Users can delete documents (removes file from disk + DB)

================================================================
              AI-POWERED FEATURES (Gemini API)
================================================================

All AI features are powered by Google's Gemini AI model
(gemini-1.5-flash) via the @google/generative-ai package.

--------------------------------------------------------------
1. SUMMARIZATION & NOTES GENERATION
   Endpoint: POST /api/ai/summarize/:docId

   - Generates a concise summary of the uploaded document
   - Extracts important keywords from the content
   - Creates structured revision notes
   - All three operations run in parallel (Promise.all)
   - Results are saved back to the document in MongoDB
   - Accessible from: SummaryPage.jsx

--------------------------------------------------------------
2. QUIZ GENERATION & EVALUATION
   Endpoints:
     POST /api/ai/quiz/:docId         — Generate quiz
     POST /api/ai/quiz/:quizId/submit — Submit answers
     GET  /api/ai/quiz/:quizId        — Get quiz by ID

   - Gemini generates MCQ and open-ended questions from the doc
   - Quiz is stored in MongoDB with the questions
   - On submission:
       * MCQ answers are auto-graded (exact match)
       * Open-ended answers are evaluated by Gemini AI
       * Score, percentage, and per-question feedback returned
   - Weak topics are identified from wrong answers using
     keyword extraction and stored for study plan generation
   - Accessible from: QuizPage.jsx

--------------------------------------------------------------
3. VIVA EXAM SIMULATION (AI Interview)
   Endpoints:
     POST /api/ai/viva/start/:docId         — Start session
     POST /api/ai/viva/:sessionId/answer    — Submit answer
     GET  /api/ai/viva/:sessionId           — View session

   - Simulates a real viva/oral exam with an AI examiner
   - Gemini generates subject-relevant viva questions
   - Student submits spoken/typed answers
   - Gemini evaluates each answer and provides:
       * Feedback on the answer given
       * The ideal/expected answer
       * A score out of 10
   - Session continues for 5 exchanges (questions)
   - After 5 questions, an overall session score is calculated
   - Previous questions are tracked to avoid repetition
   - Accessible from: VivaPage.jsx

--------------------------------------------------------------
4. FLASHCARD GENERATION
   Endpoints:
     POST /api/ai/flashcards/:docId   — Generate flashcards
     GET  /api/ai/flashcards/:docId   — Get flashcards

   - Gemini generates question-answer flashcard pairs from doc
   - Flashcards follow a spaced-repetition-friendly format
   - If flashcards already exist for a doc, they are updated
   - lastReviewed timestamp is tracked
   - Accessible from: FlashcardsPage.jsx

--------------------------------------------------------------
5. SEMANTIC SEARCH (Ask a Question)
   Endpoint: POST /api/ai/search/:docId

   - User types a natural language question about the document
   - Gemini searches the document context and returns a
     relevant, context-aware answer
   - Powered by semantic understanding, not keyword matching
   - Accessible from: SummaryPage.jsx

--------------------------------------------------------------
6. PERSONALIZED STUDY PLAN
   Endpoint: GET /api/progress/study-plan

   - Analyses the user's weak topics (from quiz mistakes)
   - Sends the weak topic list to Gemini
   - Gemini generates a tailored study plan with:
       * Study recommendations per weak topic
       * Suggested learning order
   - Requires at least one completed quiz to generate
   - Accessible from: DashboardPage.jsx / AnalyticsPage.jsx

================================================================
              PROGRESS TRACKING & ANALYTICS
================================================================

Endpoint: GET /api/progress/dashboard

Dashboard Stats (shown on DashboardPage.jsx):
  - Total number of uploaded documents
  - Total number of quizzes completed
  - Total number of flashcards generated
  - Total number of completed viva sessions
  - Average quiz score (across all quizzes)
  - Top 5 weak topics (most frequently missed)
  - 5 most recent documents
  - Last 7 quiz scores (for trend chart)

Endpoint: GET /api/progress/analytics

Detailed Analytics (shown on AnalyticsPage.jsx):
  - Full quiz history with date, score, total questions,
    and correct answers count
  - Quiz performance grouped by document (for comparison)
  - Visualized with Recharts line/bar graphs

================================================================
                     DATABASE MODELS
================================================================

USER
  - name         : String (required)
  - email        : String (required, unique)
  - password     : String (hashed, optional for Google users)
  - googleId     : String (for OAuth users)
  - avatar       : String (profile picture URL)
  - createdAt    : Date

DOCUMENT
  - userId       : ObjectId (ref: User)
  - filename     : String (stored filename)
  - originalName : String (original uploaded name)
  - extractedText: String (full text from PDF)
  - summary      : String (AI-generated)
  - keywords     : [String] (AI-extracted keywords)
  - notes        : String (AI-generated revision notes)
  - fileSize     : Number
  - mimeType     : String
  - uploadedAt   : Date

QUIZ
  - userId       : ObjectId (ref: User)
  - documentId   : ObjectId (ref: Document)
  - questions    : [{
        question, type (mcq/open), options, correctAnswer,
        userAnswer, isCorrect, feedback, score
    }]
  - totalQuestions: Number
  - score        : Number (correct count)
  - percentage   : Number (0–100)
  - weakTopics   : [String]
  - completedAt  : Date

FLASHCARD
  - userId       : ObjectId (ref: User)
  - documentId   : ObjectId (ref: Document)
  - cards        : [{ front, back }]
  - lastReviewed : Date

VIVA SESSION
  - userId       : ObjectId (ref: User)
  - documentId   : ObjectId (ref: Document)
  - exchanges    : [{
        question, answer, feedback, score
    }]
  - overallScore : Number (0–100)
  - completedAt  : Date

================================================================
                      API ENDPOINTS SUMMARY
================================================================

AUTH ROUTES (/api/auth)
  POST   /register         — Register with email & password
  POST   /login            — Login with email & password
  POST   /google           — Login/Register with Google OAuth
  GET    /me               — Get current logged-in user info

DOCUMENT ROUTES (/api/documents)
  POST   /upload           — Upload a PDF/text file
  GET    /                 — List all user documents
  GET    /:id              — Get a single document by ID
  DELETE /:id              — Delete a document

AI ROUTES (/api/ai)
  POST   /summarize/:docId         — Generate summary + notes + keywords
  POST   /quiz/:docId              — Generate quiz from document
  POST   /quiz/:quizId/submit      — Submit quiz answers for grading
  GET    /quiz/:quizId             — Retrieve quiz details
  POST   /viva/start/:docId        — Start a viva exam session
  POST   /viva/:sessionId/answer   — Submit answer in viva session
  GET    /viva/:sessionId          — Get viva session details
  POST   /flashcards/:docId        — Generate flashcards
  GET    /flashcards/:docId        — Retrieve flashcards
  POST   /search/:docId            — Ask a question about a document

PROGRESS ROUTES (/api/progress)
  GET    /dashboard         — Dashboard stats and summaries
  GET    /study-plan        — AI-generated personalized study plan
  GET    /analytics         — Detailed quiz history & performance data

HEALTH CHECK
  GET    /api/health        — Check if server is running

================================================================
                  ENVIRONMENT VARIABLES (.env)
================================================================

SERVER (.env inside /server):
  MONGO_URI              — MongoDB Atlas connection string
  JWT_SECRET             — Secret key for signing JWTs
  GEMINI_API_KEY         — Google Gemini API key
  FIREBASE_PROJECT_ID    — Firebase project ID
  FIREBASE_CLIENT_EMAIL  — Firebase service account email
  FIREBASE_PRIVATE_KEY   — Firebase service account private key
  PORT                   — Server port (default: 5000)

CLIENT (.env inside /client):
  VITE_API_URL           — Backend API base URL
  VITE_FIREBASE_*        — Firebase client config values

================================================================
                    HOW TO RUN THE PROJECT
================================================================

1. INSTALL DEPENDENCIES
   Root:         npm install
   Server:       cd server && npm install
   Client:       cd client && npm install

2. START THE BACKEND
   cd server
   npm run dev         (development with nodemon)
   npm start           (production)

3. START THE FRONTEND
   cd client
   npm run dev         (opens on http://localhost:5173)

4. ENSURE ENVIRONMENT VARIABLES ARE SET
   - /server/.env  →  MONGO_URI, JWT_SECRET, GEMINI_API_KEY,
                       FIREBASE credentials
   - /client/.env  →  VITE_API_URL, VITE_FIREBASE_* values
